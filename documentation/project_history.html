Main Use Cases for Project DEW

Project definition

DEWLINUX is a Terminal-only System Profile on top of a Debian-based Linux Environment. The system is designed to provide a satisfactory level of comfort for new terminal users.

Core ideas

I came up with the idea while watching the Fallout (2024) tv-series. Here is how the focus of my idea evolved over time:

  1. Mimic the Vault Overseer computer as a modern terminal-only system.
    • CORE PROJECT DEW SYSTEM DESIGN SPECIFICATION:
      1. Virtual machine as a main platform, since this would be just an idea testing
      2. Debian 13 as a software platform
      3. Keeping it minimalistic by using the most basic terminal possible (8 colors)
      4. While the system would have non-stop network connectivity, keep the email setup OFF-LINE based out of a nostalgic vibe
      5. Despite it's a terminal-based system, provide the most of the user experience technically possible for the chosen terminal type (!)
  2. But what is possible to be done with such a system in 2026? What the user experience would be?
  3. Should I make my own Linux distro? (Nah!)
  4. Ok, but let's not just make a terminal system, let's actually make it ready to use, by providing installation and setup automation
    • Let's provide a wizard-like GMAIL setup.
      • OPS! Seems the major providers are no longer using the familiar "username" and "password" IMAP setup. And just like 2-3 months ago all major providers decided to phase out the so called "app passwords". What should I do?
      • The only alternative is OAUTH2. Let's be heroes and do an OAUTH2 setup.
      • But setting up OAUTH2 is real s#it for a person never exposed to GCP. And all the available solutions for some reason did not worked on my system. I want to QUIT!!
      • I can't quit. I went hardcore and coded my own wizard. It's actually not hard at all, once you get the logic of things.
      • With regards to the 90s DOS era, let's make the email setup entirely off-line based! User would run a shell script to sync the email when they want to. And if they wish so - they could always make a cron-job out of it.
  5. With regards to the 90s pure DOS user experience, let's explore and see how I could improve the terminal-only user experience by providing the user with curated, tested and quality-assured set of terminal enhancements.
  6. Let's make it a user-friendly terminal user experience introduction for new terminal users.
  7. My personal terminal UX test ground with the idea that I could adopt some of the terminal tools I tested.

Main Use Cases for the design of Project DEW

DISCLAIMER: Since Project DEW was created as a playground of this idea I've got, it was designed to run on a virtual machine from the very beginning. For this reason some things were never taken into consideration, just like document printing for example.

  • Having some colors for comfort (similar to 4DOS)
  • Files management
  • Using internet websites
  • Using the most common email - GMAIL with an email client
  • Viewing digital content:
    • Basic text files (ASCII files)
    • Microsoft Office Word and Excel documents
    • Other spreadsheet formats
    • Images
    • Video viewing, maybe watching movies
    • Music listening
  • Creating digital content (authoring):
    • Basic text files (ASCII files)
    • Word processing - CURRENTLY NOT MATURE
    • Creating a spreadsheet
    • Creating Microsoft Office Word and Excel documents - CURRENTLY UNAVAILABLE!
    • Image creation and manipulation - CURRENTLY UNAVAILABLE!
    • Video editing - CURRENTLY UNAVAILABLE!
    • Creating music - CURRENTLY UNAVAILABLE!
  • Software development
    • Perl
    • Python 3
    • Lua
    • C/C++
    • Go (Golang)
    • Rust
    • gasm, nasm
  • Self-organization (TODO management etc)
